Network Intrusion Detection (IDS) Lab

Project Overview

The project shows how to build an IDS using Snort software through a virtualized laboratory demonstration. The system monitors network traffic between a Windows Server and client machine, detecting suspicious activity and generating alerts in real time. The laboratory environment simulates a small business network and teaches students how to monitor network security and analyze network traffic and identify potential security threats.
The project was completed as part of independent learning and practical skill development in network security system administration and cybersecurity fundamentals.
________________________________________
Objectives

• The Linux virtual machine will function as a Snort sensor through its deployment. 

• The system will observe network activities between the Windows Server and client virtual machine. 

• The system will use specific Snort rules which I developed to identify suspicious and harmful behaviors. 

• The IDS will undergo testing by simulating network attacks which include ping floods and port scans. 

• The network activities and alerts will be documented for analytical and reporting purposes. 

• The system enables users to learn about traffic monitoring and the process of detecting intrusions. 
________________________________________
Technologies Used

• Snort serves as an industry-standard IDS which provides a free and open-source solution. 

• The system operations use a Linux virtual machine which supports either Ubuntu or Kali Linux. 

• Oracle VirtualBox serves as the virtualization software platform. 

• The system uses Windows Server 2022 to replicate a Domain Controller from Phase 1 laboratory simulation. 

• Network monitoring and logging tools use Snorby and ELK Stack for dashboard functions. 

• The system uses Draw.io to create and display network mapping functions. 
________________________________________
Lab Environment Configuration

• The Linux VM has been assigned 4 GB of RAM. 

• The system has been assigned 2 CPU cores. 

• The virtual machine has a 40 GB virtual hard disk capacity. 

• The system uses NAT configuration for Network Adapter 1 to provide internet connectivity. 

• The Network Adapter 2 of the system operates as an Internal Network to enable traffic monitoring between virtual machines. 

• The IDS sensor sits within the internal network of the Windows lab so it can capture all network traffic. 

• The system uses custom Snort rules to identify suspicious behaviors which include multiple ICMP requests and port scanning activities.

Key Tasks Performed

1. The team implemented a Linux virtual machine through Oracle VirtualBox installation. 
2. The team established Snort as a network intrusion detection system through the installation and configuration process. 
3. The writing process produced specific Snort rules designed to identify network activities which include ICMP ping floods. 
4. The IDS was tested through network behavior simulation from the client virtual machine which included port scanning and pinging efforts. 
5. The team captured Snort alert logs which they used for log analysis. 
6. The document shows all network activities and security events which were triggered by the system rules. 
7. The team created a network diagram which displays the IDS sensor location throughout the virtual environment.
